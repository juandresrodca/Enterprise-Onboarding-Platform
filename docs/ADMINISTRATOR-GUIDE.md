## Dashboard

Counters (total users, created last 7 days, pending onboarding jobs, errors
last 24 h), license consumption bars, recently created users, live audit feed
and recent jobs.

## Create users

1. Enter how many users (1–50) → **Generate forms**.
2. Each collapsible card collects the full attribute set. Leave *Display
   name / Username / UPN / Email* empty to have them derived
   (`first.last` convention; collisions get a numeric suffix with a warning).
3. **Organizational unit** — *Browse* opens the OU tree picker.
4. **Manager** — type ≥2 characters for directory autocomplete.
5. **Groups** — *Select groups* opens the picker (search + filter by
   security / distribution / M365).
6. **Licenses / mailbox / shared mailboxes** — remaining license counts are
   shown; exhausted pools produce warnings at validation.
7. **Password** — generated by default (policy-compliant, shown once after
   execution). Manual passwords are validated against the policy, including
   the no-name-parts rule.
8. **Validate** → issues appear inline on the exact field and card.
9. **Preview & execute** → review the full execution plan → **Approve**.
10. Watch the live progress window (progress bar, streaming logs, per-user
    results). Copy generated passwords now — they are never stored. Deliver
    them through your secure channel (e.g. LAPS-style envelope, password
    manager share), never email.

## Copy existing user (clone)

1. Search the template employee (e.g. *John Smith*) and select them; the
   summary chip shows groups/licenses/shared-mailbox counts.
2. Untick any attribute family you don't want copied (OU, organization,
   manager, address, office phone, groups, licenses, shared mailboxes,
   proxy-address pattern, extension attributes, home folder, logon script).
   Identity fields — SID, GUID, password, username, email, employee ID,
   display name, mobile — are **never** copied, by design.
3. Enter first/last name (and optional title/employee ID) for each new user.
4. **Preview clone** shows the merged result; approve to execute.

## Bulk import

- Formats: **CSV** (comma or semicolon), **Excel** (.xlsx, first sheet),
  **JSON** (array or `{"users": [...]}`). Max 200 rows / 5 MB.
- Download the CSV template from the page. Column headers are matched
  loosely — `Given Name`, `Surname`, `Reports To`, `MemberOf`,
  `License Type`, `Organizational Unit`… all map automatically.
- Multi-value cells (groups, licenses, shared mailboxes) use `;` or `|`.
- **Quote OU distinguished names in CSV** — they contain commas.
- After parsing, the full validation report appears per row; execution goes
  through the same preview/approval gate as manual creation.

## Audit logs

Filter by action, status, actor, target; export respects the active filters.
Every entry records timestamp (UTC), actor, role, action, target, status,
computer, source IP and structured details (e.g. exact groups assigned).

## Settings

- **Your session** — role, permissions, expiry.
- **Platform** — environment, Entra ID status, domain, UPN suffix, naming
  convention, home-folder share, session timeout.
- **Password policy** — Global Admins can adjust length/complexity/age;
  changes take effect immediately for validation and generation and are
  audited (`settings.update`).

## Operational notes

- **Session timeout**: 30 min sliding (configurable). Work in progress on a
  form is client-side only — validate early.
- **Hybrid environments**: cloud steps (licenses, mailboxes) depend on Azure
  AD Connect sync; the job engine records per-step warnings (e.g. "not
  synced yet") rather than failing the account creation.
- **Passwords**: the platform never stores or logs any password. If a
  generated password is lost, reset it in AD — it cannot be recovered.
